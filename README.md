## ⚡ Quick Start (5 Minutes)

### Step 1 — Download
```bash
git clone https://github.com/YOUR_USERNAME/job-automation-studio.git
cd job-automation-studio
```

### Step 2 — Install Requirements
```bash
pip install gradio PyYAML
```

### Step 3 — Run the Dashboard
```bash
python app.py
```
Then open **http://localhost:7860** in your browser.

### Step 4 — Answer the Questions
- **Step 1:** Tell us your name, email, CV, target job
- **Step 2:** Pick your locations, salary, companies to skip
- **Step 3:** Choose job sites and tools

### Step 5 — Download & Run
1. Click **"Generate My Script"**
2. Download the `.py` file
3. Give it to your AI agent:
   > *"Run my job automation script at /path/to/your-script.py"*

## 🛠️ How to Run the Script

### Option A — 🤖 With AI Agent (RECOMMENDED — Easiest)

This is the **best experience**. Your AI agent handles everything:

> Just say to your AI: **"Run my job automation script"**

The AI will:
- ✅ Open the job search pages
- ✅ Log into job sites with your credentials
- ✅ Fill in application forms with your details
- ✅ Write personalised cover letters per job
- ✅ Handle anti-bot blocks (error_code:9101 on LinkedIn)
- ✅ Skip banks, senior roles, companies you don't want
- ✅ Track what you've applied to

**Requires:** QClaw, OpenClaw, or any AI agent with browser control.

---

### Option B — 🖥️ With Playwright (Free, No AI)

```bash
pip install playwright
playwright install chromium

# Then in your script, uncomment the Playwright section
python your_script.py
```

**Pros:** Free, reliable browser automation
**Cons:** No AI smarts — form filling is basic

---

### Option C — 🌐 With Selenium (Classic)

```bash
pip install selenium
# Download ChromeDriver: https://chromedriver.chromium.org/
# Add it to your system PATH

python your_script.py
```

**Pros:** Widely used, lots of tutorials
**Cons:** More setup, can be detected as bots

---

### Option D — 🤲 Manual (Safest)

Use the **Search URLs** generated by the dashboard. They'll open the right job searches in your browser. Apply manually — slower, but zero risk of being blocked.

## 🤖 Why Does This Need an AI?

This tool **generates** the automation script. Actually *running* it well requires an AI agent because:

- **Anti-bot detection:** LinkedIn and other sites actively block automated tools. An AI can recognise and handle `error_code:9101` (LinkedIn's "suspicious activity" block), adapt when buttons move, and decide when to pause.
- **Dynamic UI:** Job sites change their layout constantly. An AI can read the page and figure out what to click next.
- **Smart form filling:** Different jobs have different application forms. An AI can parse each form and fill it intelligently.
- **Personalised cover letters:** One generic cover letter is weak. An AI can write a short personalised note per application.
- **Error recovery:** When something goes wrong, an AI can decide what to do next instead of just stopping.

**Think of it as:** The script is the blueprint. The AI is the builder.

## ⚠️ Common Errors & Fixes

| Error | Cause | Fix |
|---|---|---|
| `error_code:9101` | LinkedIn blocked you | Wait 2-3 minutes, run during off-peak hours |
| `File not found` | Script moved | Re-download from the dashboard |
| `No Easy Apply button` | Job doesn't support it | Try a different job listing |
| Script stopped after 5 jobs | LinkedIn rate limit | Lower daily cap to 20-30, use VPN |
| UI changed (buttons missing) | Site updated their design | Re-generate the script |
| Captcha appeared | Suspicious activity | Use a VPN or wait an hour |

## ⚙️ Customisation

### Adding a new job site
Edit `SITES/directory.yml`:
```yaml
my_site:
  name: "My Job Site"
  url: "https://myjobsite.com"
  search_pattern: "/jobs?q={role}&l={location}"
  easy_apply: true  # or false
```

### Changing filters
Edit your generated script's CONFIG section at the top:
```python
SKIP_SENIOR  = True    # Set to False to include senior roles
REMOTE_ONLY  = True    # Set to False to include on-site jobs
DAILY_CAP    = 50      # Max applications per day
EXCLUDE      = ["bank", "fintech", "my old company"]
```
